## Flow 흐름 정리

## 유저
### 회원 가입
1. 회원 가입 요청 request 검증
2. 이메일 중복 검증
3. (비즈니스 로직) 비밀번호 암호화, 기본 권한(user, admin) 설정
4. 회원 가입 / 회원 가입 실패

### 회원 목록 조회
1. 회원 목록 조회 요청 request 검증
2. 목록 조회 성공 (`빈 리스트`)

### 회원 상세 조회
1. 회원 상세 조회 요청 request 검증 (`memberId`)
2. 회원 존재 여부 검증
3. (비즈니스 로직) 하나의 회원 정보 조회
4. 회원 상세 정보 반환 / 실패

### 회원 탈퇴
1. 회원 탈퇴 요청 request 검증 (`memberId`, `비밀번호`)
2. 유저 유효성 검증 (존재 여부)
3. (비즈니스 로직) 회원 탈퇴 처리(하드 딜리트)
4. 회원 탈퇴 완료 / 실패

---

## 상품

### 상품 등록
1. 상품 등록 요청 request 검증
2. 유저 유효성 검증 (권한 및 등록자 확인)
3. (비즈니스 로직) 상품명 중복, 카테고리/할인 여부, 이미지/옵션/조합 및 재고 저장 
4. 상품 등록 / 상품 등록 실패

### (+ 카테고리)
1. 의류
2. 디지털/가전
3. 가구/인테리어
4. 식품
5. 패션잡화
6. 스포츠/레저

### 상품 목록 리스트 조회
1. 상품 목록 조회 요청 request 검증
2. 목록 조회 성공 (`빈 리스트`)

### 상품 상세 조회
1. 상품 상세 조회 요청 request 검증 (`productId`)
2. 상품 유효성 검증 (존재 여부)
3. (비즈니스 로직) 하나의 상품 조회
4. 상품 상세 정보 반환 / 실패

### 상품 수정
1. 상품 수정 요청 request 검증 (`productId`)
2. 유저 유효성 검증 (권한 및 등록자 확인)
3. 상품 유효성 검증 (존재 여부)
4. (비즈니스 로직) 상품명 중복, 카테고리/할인 여부, 이미지/옵션/조합 및 재고 수정
5. 상품 수정 / 상품 수정 실패

### 상품 삭제
1. 상품 삭제 요청 request 검증 (`productId`)
2. 유저 유효성 검증 (권한 및 등록자 확인)
3. 상품 유효성 검증 (존재 여부)
4. 상품 삭제 / 상품 삭제 실패

---

## 주문

### 주문 생성
1. 주문 생성 요청 request 검증 (`memberId`, `productId` 등)
2. 유저 유효성 검증 (로그인 여부)
3. 상품 유효성 검증 (재고, 판매중 여부)
4. (비즈니스 로직) 주문 금액 계산, 재고 수정, 결제 요청 및 승인
5. 주문 생성 성공 / 실패

### 주문 조회
1. 주문 조회 요청 request 검증 (`orderId`)
2. 유저 유효성 검증 (본인 주문인지)
3. (비즈니스 로직) 주문 존재 여부 확인, 주문 정보 조회 등
4. 주문 조회 성공 / 실패
